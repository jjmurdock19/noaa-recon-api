noaa-recon-api — raw netCDF passthrough demo
This page proves the netcdf-three integration end-to-end: it fetches a netCDF file, decodes a 3D variable in the browser, and volume- renders it with Three.js/WebGL — no server-side image processing.

It defaults to the sample dataset bundled with the netcdf-three vendor submodule (a generic, non-geospatial volume viewer — see README.md "netcdf-three" section for why). Once GET /v1/raw/netcdf is implemented (follow-up phase, see app/routers/raw.py), swap DATA_URL below for a real GOES/TDR subset from the running API.
